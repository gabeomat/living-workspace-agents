# First-Run Onboarding — "what do you sell, and how do you sound?"

Run this the first time, before any sequence work. It's an interview, not a form — a few questions at a time, confirm back what you learn. Check their `CLAUDE.md`/Brain first and **don't ask what the workspace already answers.** At the end, write their real profile, seed the story bank, and build the `nurture/` structure.

## 1. The offers — "what can an email sell?"

For each offer: name, one-line promise, **real price**, and the buy path (checkout link? application? call booking?). Sequences only ever sell and quote these. Also capture the offer *ladder* if one exists — what's the natural next thing after each purchase (feeds post-purchase sequences).

## 2. The list — "who are you writing to, and from where?"

- **Platform** — which email tool (any works: MailerLite, Kit, Mailchimp, GoHighLevel, ActiveCampaign…). Is it connected to Claude (MCP/connector)? Note the answer either way — delivery adapts; don't set a connection up now.
- **List state** — size (roughly), how engaged, when they last heard from the user. A list that's been quiet 6 months changes the first build recommendation (a re-engagement sequence likely comes before anything else — say so if true).
- **How people arrive** — opt-in pages, events, purchases, manual adds. This is the funnel map: each arrival point is a sequence this agent will eventually own.
- **Sender identity + compliance footer** — the from-name, and the business/address/unsubscribe block their platform appends (or needs).

## 3. The voice

- Brand-voice skill installed? Route drafting through it.
- If not: 2–3 samples of emails *they've actually sent and liked* (the exact register needed), plus voice rules (sign-off, things they'd never say).
- **Sending-cadence comfort** — some owners flinch at daily emails during a close. Find the line now, not mid-launch; the architecture respects it (and can push back once, gently, with the reasoning — then respect it).

## 4. Seed the story bank (the heart of the session)

Run the elicitation questions in `references/story-bank-spec.md` — get **3–5 real stories**, tagged, with the "usable publicly?" answer for each. This is the wow moment of onboarding: most owners have never seen their own stories organized as an arsenal. Reflect it back: *"you now have proof for [these beliefs] — and a gap at [this one], so keep an eye out for that story."*

## 5. Build and confirm

1. Create the workspace: `nurture/stories/story-bank.md` (seeded) + `nurture/sequences/` (empty).
2. Copy `profiles/_TEMPLATE.md` → `profiles/<their-business>.md`, fill from the interview, read it back in two sentences, correct anything.
3. Tell them the first move: **"tell me what funnel you're building"** — and if their answers surfaced an obvious first build (an event next week with no follow-up sequence; a list gone cold), name it as the recommended starting point.
