# Follow-Up & Proposal — the drafting formulas

Both drafts run on the mid tier, in the user's voice (through their brand-voice skill if installed), and both end at presentation — **the user sends.** Both obey the same integrity rules: every remembered detail traces to the transcript or deal file; every price traces to the profile.

## The follow-up (Mode 1, step 3)

**The standard: drafted immediately, approved-and-sent within the hour of the call.** Speed is the feature — a same-hour follow-up that quotes the conversation back accurately is rarer than a good pitch, and it's what being *seen* feels like from the buyer's side.

Structure (short — this is an email, not a report):

1. **Open with their thing, not ours.** Reference the most alive moment of the call — a goal, a laugh, a problem named — in their words. Never "Great chatting with you!"
2. **Confirm what was agreed** — the next step, dated, both directions: "You're sending the team headcount; I'll have the proposal to you Thursday."
3. **Deliver anything promised for now** (a link, an answer they asked for) — arriving with the goods an hour later is the credibility move.
4. **One confident line toward the next step.** Not a re-pitch. If the call ended warm but fog-bound (no next step set — it happens), the follow-up is where the fog gets fixed: propose a specific time.

Tone rules: warm, brief, zero confidence leaks (see the rubric's four patterns — check the *draft* for them, not just the call). No "just checking in," no "sorry to bother," no re-selling what was already sold.

**Later nudges** (from a pipeline check, when a deal goes quiet): reference the real open loop ("You mentioned wanting this settled before the fall season — is that still the plan?"), offer something useful, one clear question. Respect the profile's [[Follow-Up Window]]; after two unanswered nudges, recommend the honest close-the-loop message ("Sounds like now isn't the time — I'll leave this with you") and, if lead-manager is installed, hand the contact across for long-term nurture.

## The proposal (Mode 3)

Only when a call actually reached "send me a proposal" (or the user explicitly asks). A proposal built from the prospect's own recorded words closes what a template never will — the reader should feel *heard on page one.*

Skeleton:

1. **Where you are, in your words.** Their situation and goal, built from the "Their words" bank — quoted or faithfully tight-paraphrased. This section does most of the selling.
2. **What it's costing you.** The stakes *they* named — money, time, stress — from the transcript. Never invent stakes they didn't claim.
3. **What I'm proposing.** The offer, scoped to what was actually discussed on the call. Plain language, concrete deliverables, what happens in week one.
4. **What it costs.** The price from the profile — stated flat, once, with payment terms. **Never invent a price, never pad one, never offer an unprompted discount.** If the call discussed a custom scope with no profile price, mark the number `[for you to set]` and say so out loud rather than guessing.
5. **What happens next.** One step, one date: "Reply yes and we start Monday" / "I'll call Tuesday at 2 to answer questions."
6. **Objections, pre-answered (optional).** If the call surfaced a hesitation, address it plainly in one short section — acknowledging their stated concern reads as listening, not defensiveness.

Length: default 1–2 pages. Confident throughout — the four leak patterns apply doubly in writing, where a hedge sits on the page forever. No fake urgency, no invented scarcity; real deadlines only if the user confirms they're real.

**Format:** default is a clean markdown/text draft the user can paste anywhere. If the environment has document-generation skills (PDF, docx) or the user has a branded template, offer to render it there — degrade gracefully to plain text.
