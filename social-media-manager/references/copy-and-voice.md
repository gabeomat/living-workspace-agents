# Copy & Voice — caption stages

How the source content becomes captions, and how those captions get the user's voice. The voice itself
is NOT defined here — it's read from the user's **brand-voice skill**. This file is the process; the voice
skill is the specifics.

## Stage 1 — Caption writing (from the source)

1. Read the source content (transcript, video, blog post, notes). Extract distinct **teachable beats** —
   insights, reframes, stories, steps, quotable lines.
2. You need enough beats to fill every post in the profile's mix. Map beats → posts:
   - **Statics** — one sharp idea each (a claim + a supporting line; or a short numbered framework; or a quote).
   - **Carousels** — one theme per carousel, broken into a cover hook + numbered content beats + a CTA.
   - **Reel** — one narrative through the profile's frame arc (e.g. Hook → Tension → Interrupt → Payoff → CTA).
3. Pull CTAs from the user's **Business Brain** (real offers / current priority). If the Brain isn't readable,
   use the profile's standing CTA or ask the user for this week's CTA. Never invent an offer.
4. If the source is too thin to fill the mix honestly, mine sub-points first; if it still can't, tell the user
   rather than padding with filler. A shorter honest week beats a padded one.

## Stage 2 — Voice pass

1. Load the user's **brand-voice skill** and rewrite every caption through it — apply their strip rules
   (what to cut) and keep rules (what to protect).
2. Apply the voice pass to every piece: static captions, carousel slides, Reel frames, and the Saturday caption.
3. **If no brand-voice skill exists:** apply a neutral pass — clear, direct, human, specific, no filler, no
   hedging, no hype — and tell the user plainly that building a brand-voice skill is the single biggest upgrade
   to output quality. Don't fake a voice you can't read.

### Neutral fallback voice (only when no voice skill is present)
- Lead with the point, not a warm-up.
- Concrete over abstract — real specifics, not vague claims.
- No filler transitions ("at the end of the day," "the thing is").
- State CTAs directly ("link in bio," "DM me X") — no "you might want to."
- Keep it human, not corporate. Short sentences.

## Stage 2.5 — Review gate (HARD STOP)

Present ALL copy in one message, grouped by day/post:
- each static: caption + on-card text
- each carousel: caption + every slide's text (cover, numbered beats, CTA)
- the Reel: every frame's text + the post caption

Then **STOP.** Build nothing until the user approves. "Go / approved / continue" = proceed. Any edits →
revise, re-present only the changed pieces, wait again. This gate is non-negotiable — it's what keeps the
user in control of their brand before production happens.

## Notes
- On-card text ≠ caption. The card shows a short headline/quote; the caption is the longer post text. Write both.
- Keep card text tight — it has to fit the layout. Long thoughts go in the caption, not on the card.
- Match CTA to the profile's current priority so the week drives toward one thing, not five.
