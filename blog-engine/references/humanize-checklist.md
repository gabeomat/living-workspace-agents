# Humanize Checklist (Step 4)

After the draft is written, switch to editor mode. Job: make sure nothing reads as AI and every line sounds like the **user's voice**. First load the voice source — a voice skill, a saved profile's voice section, or the user's brand/voice docs. If none exists, infer the voice from the user's existing writing in the workspace. Don't add new SEO; protect voice, strip tells, then confirm the SEO survived.

## Strip the AI tells (universal)
- [ ] **Em-dashes / en-dashes: zero.** Replace with commas, periods, colons, parentheses. Hyphens in compound words are fine. (Single most important pass — it's the #1 AI tell.)
- [ ] **Throat-clearing intros gone:** no "In today's world," "In this post we'll explore." Open with the answer + relevance.
- [ ] **Filler transitions gone:** "Moreover," "Furthermore," "In conclusion," "At the end of the day," "It's important to note that."
- [ ] **Hedge-stacking gone:** not every sentence needs "may," "might," "can sometimes." Say it plainly where it's true.
- [ ] **Robotic parallelism softened:** vary sentence length and shape. Real writing has rhythm.
- [ ] **Over-cheerful uniformity gone:** grounded, not a relentless pep rally.
- [ ] **Keyword stuffing removed:** if the primary keyword reads forced anywhere, rephrase.
- [ ] **"Not X, it's Y" antithesis rationed:** at most once, only if it's a genuine payoff line. (It's a common AI overuse pattern.)

## Lock in the user's voice
Check the draft against the user's voice source. It should:
- [ ] Sound like a specific real person with a point of view, not a content mill or a textbook.
- [ ] Hit the user's signature moves and tone (whatever their voice source defines — e.g. a warm, validate-then-challenge therapist voice, or a direct, shift-first, anti-hype operator voice). Read the voice source for the exact rules.
- [ ] Respect the user's hard bans (retired hooks, banned phrases, punctuation limits — whatever they've specified).
- [ ] Open in a way that earns the reader's attention in the first paragraph.

## Trust + safety pass
- [ ] Claims are accurate; anything factual / clinical / financial is cited to a real source where needed.
- [ ] **If the user's topic is YMYL** (health, finance, legal, relationships): no guarantees or outcome promises, author authority / credentials explicit, and a light caring nudge toward professional help on heavy topics.
- [ ] Author / brand authority is clear per the user's setup.

## SEO-integrity pass (don't let the edit break ranking)
- [ ] Primary keyword still in: title, first ~100 words, >=1 H2, meta description, slug.
- [ ] H2s are still question-shaped.
- [ ] FAQ block intact with clean 2–4 sentence answers.
- [ ] Internal links + CTA still present and natural.
- [ ] Extractable answer still sits near the top, self-contained.

If all passes clear, deliver. If any voice section still feels flat, rewrite that section by hand rather than tweaking words.
